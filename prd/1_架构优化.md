# 架构优化 PRD：统一运行框架（botcore.Engine）

> 目标：让平台层与业务层边界清晰、运行流程丝滑可复用，并可扩展到更多平台。
> 
> 版本：v2.0（2026-01-28 决策确认版）

---

## 1. 背景与问题

### 1.1 当前现状
- `platform/wecom` 的 `Bot` 承担了"协议处理 + 业务编排 + 流式会话管理"
- `botcore` 仅提供接口/工具，但没有统一运行框架
- 整体架构主干不清晰

### 1.2 核心理念
- **平台层（通信层）**：只负责协议与传输
- **业务层（Command + LLM）**：只负责处理与结果

---

## 2. 目标与约束

### 2.1 目标
1. 新增 `botcore.Engine` 作为统一运行框架
2. **新增 `CommandInput` 替代 `FirstSnapshot`** 作为统一业务输入
3. 平台 Bot 负责协议与流式，Engine 负责生命周期编排
4. 保持 channel 返回模式

### 2.2 约束
- Engine 放在 `pkg/botcore/`
- Engine 不接触 HTTP、不管理流式会话
- Engine 作为可选包装层，保持向后兼容

---

## 3. 总体方案

### 3.1 架构图

```
                         ┌───────────────────────────────┐
HTTP Callback (WeCom) -> │ wecom.Bot (通信层)             │
                         │ - HTTP 解析/签名校验/解密       │
                         │ - ParseInput -> CommandInput   │
                         │ - StreamManager (ID = StreamID)│
                         └───────┬───────────────────────┘
                                 │ CommandInput
                                 v
                         ┌───────────────────────────────┐
                         │ botcore.Engine (编排层)         │
                         │ - Hook: Before/After/OnError   │
                         │ - 调用 PipelineInvoker         │
                         └───────┬───────────────────────┘
                                 │ <-chan StreamChunk
                                 v
                         ┌───────────────────────────────┐
                         │ command.Manager (业务层)        │
                         │ -> Cobra 命令路由 / LLM 调用    │
                         └───────────────────────────────┘
```

// pkg/platform/wecom/bot.go

func (b *Bot) ParseInput(msg *Message) botcore.CommandInput {
    session, _ := b.StreamSessions.CreateOrGet(msg)
    
    var media []botcore.MediaItem
    if msg.Image != nil && msg.Image.Base64 != "" {
        media = append(media, botcore.MediaItem{
            Type: botcore.MediaTypeImage,
            Image: &botcore.MediaImage{
                Base64: msg.Image.Base64,
                MD5:    msg.Image.MD5,
            },
        })
    }

    return botcore.CommandInput{
        ID:       session.StreamID,  // 统一使用 StreamID
        SenderID: msg.FromUserName,
        ChatID:   msg.FromUserName,  // 根据实际情况调整
        ChatType: botcore.ChatTypeSingle,
        Text:     msg.Content,
        Raw:      msg,
        Media:    media,
    }
}

### 4.3 Engine 接口

```go
// pkg/botcore/engine.go

type Engine struct {
    pipeline PipelineInvoker
    hooks    Hooks
    buffer   int
    logger   *log.Logger
}

type Hooks struct {
    Before  func(ctx context.Context, input CommandInput) error
    After   func(ctx context.Context, input CommandInput)
    OnError func(ctx context.Context, input CommandInput, err error)
}

func (e *Engine) Handle(ctx context.Context, input CommandInput) <-chan StreamChunk {
    out := make(chan StreamChunk, e.buffer)
    
    go func() {
        defer close(out)
        
        // Hook: Before（可阻断）
        if e.hooks.Before != nil {
            if err := e.hooks.Before(ctx, input); err != nil {
                if e.hooks.OnError != nil {
                    e.hooks.OnError(ctx, input, err)
                }
                out <- StreamChunk{Content: err.Error(), IsFinal: true}
                return
            }
        }
        
        // 转换为 FirstSnapshot 调用现有 Pipeline（兼容期间）
        snapshot := input.ToFirstSnapshot()
        innerCh := e.pipeline.Trigger(snapshot, input.ID)
        
        // 转发 + context 取消监听
        for {
            select {
            case <-ctx.Done():
                return
            case chunk, ok := <-innerCh:
                if !ok {
                    if e.hooks.After != nil {
                        e.hooks.After(ctx, input)
                    }
                    return
                }
                out <- chunk
            }
        }
    }()
    
    return out
}
```

## 5. Bot 接入方式

```go
type Bot struct {
    Engine   *botcore.Engine        // 可选
    Pipeline botcore.PipelineInvoker // 兼容旧模式
}

func (b *Bot) Initial(msg *Message, ...) (EncryptedResponse, error) {
    input := b.ParseInput(msg)
    
    var outCh <-chan botcore.StreamChunk
    if b.Engine != nil {
        outCh = b.Engine.Handle(ctx, input)
    } else if b.Pipeline != nil {
        outCh = b.Pipeline.Trigger(input.ToFirstSnapshot(), input.ID)
    }
    
    // ... 首帧处理逻辑
    go b.consumePipeline(outCh, msg.MsgID, input.ID)
}
```
